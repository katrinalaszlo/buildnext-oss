# Cross-Team Velocity Is Guesswork

**Status:** proposed
**Type:** feature-request
**Evidence:** 1 customer

## User Story

> As a VP Engineering I want a single reliable velocity metric per team per sprint so that I can make staffing decisions without spending hours reconciling spreadsheets

## Acceptance Criteria

- [ ] One number per team per sprint, updated automatically from Linear/GitHub/PagerDuty
- [ ] Trend view showing velocity over time per team
- [ ] Flag when a team's unplanned work exceeds a threshold (e.g. 50% of sprint)
- [ ] Team leads can see their own data without VP intermediation
- [ ] No manual data entry required — API integrations only
