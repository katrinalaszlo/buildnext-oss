How it works

  1. Clone the repo and copy the example config.
    git clone https://github.com/katrinalaszlo/buildnext-oss.git
  2. Drop a transcript into the raw/ folder. Plain text or markdown.
  3. Tell your AI editor to "ingest." Works with Claude Code, Cursor, or Codex. The agent reads the schema and does the rest.

What you get

  • Customer pages with exact quotes, speaker attribution, and evidence tags
  • User stories with acceptance criteria, grounded in evidence from multiple customers
  • Feature pages that group stories by product area, problem-first
  • Obsidian graph view showing how customers, stories, and features connect
  • Full control — edit any file and the agent respects your changes
